{% markdown %} Calcite Web's grid is a float-based grid system that utilizes viewport width to determine sizing. The grid can be [configured with SASS](../../documentation/grid/#configuration) and is by default responsive. This makes creating layouts that work across multiple screen sizes much easier and less time consuming. Read on for some quick pointers on constructing layouts with the Calcite Web grid classes. ## 1. Always use a grid container All grid layouts should be wrapped in an element with the [`grid-container` class](../../documentation/grid/#container). The grid container will take care of setting the max width, centering the layout, clearing the floats, and compensates for screen realestate taken up by scroll bars.

## 2. Size elements with column classes The `column-*` classes are the main mechanism provided by Calcite Web to size elements horizontally. The grid by default is twenty-four columns on desktop, twelve on tablet, and size on mobile. Using the [responsive column classes](../../documentation/grid/#responsive-columns) you can specify different sizes for different viewports. Columns can also be nested inside one another. In certain instances you may need to use the [first and last classes](../../documentation/grid/#first-and-last) to fix the left-hand alignment of a nested column. ### Examples
