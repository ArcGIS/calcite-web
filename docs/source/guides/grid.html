{% markdown %} Calcite Web's grid is a float-based grid system that utilizes viewport width to determine sizing. The grid can be [configured with SASS](../../documentation/grid/#configuration) and is by default responsive. Read on for some quick pointers on constructing layouts with the Calcite Web grid classes.

1. Always use a grid container

All grid layouts should be wrapped in an element with the [`grid-container` class](../../documentation/grid/#container). The grid container will take care of setting the max width, centering the layout, clearing the floats, and compensates for screen realestate taken up by scroll bars.

## 2. Size elements with column classes The `column-*` classes are the main mechanism provided by Calcite Web to size elements horizontally. The grid by default is twenty-four columns on desktop, twelve on tablet, and size on mobile (see below for more on responsive classes). Columns can also be nested inside one another. In certain instances you may need to use the [first and last classes](../../documentation/grid/#first-and-last) to fix the left-hand alignment of a nested column.

## 3. Use the phone and tablet classes for responsive designs The column classes will automatically resize themselves on phone and tablet, but these defaults are just guesses, and many times you'll need to specify layouts for specific device form factors. Using the [responsive column classes](../../documentation/grid/#responsive-columns) you can specify different sizes for different viewports. ## 5. Use block-group to create gridded layouts [Block-groups](../../documentation/grid/#block-groups) are meant for situations where you'd like a regular gridded layout with a known number of columns. They use an `inline-block` based grid that is _not_ based on viewport-width. This means you can use them inside any containing element, no matter the size. To create a block group, add the `block-group` and `block-group-X-up` classes to the containing element as well as the `block` class to each child item.
